Justificativas dos índices criados no database Somativa

Coleção products
1. { categoryId: 1 }
Motivo: consultas que listam produtos por categoria são muito comuns, como mostrar todos os produtos da categoria “Celulares”.
Benefício: acelera buscas com find({ categoryId: ... }), evitando varredura completa da coleção.

2. { sellerId: 1 }
Motivo: relatórios ou dashboards de vendedores precisam listar apenas os produtos de um determinado vendedor.
Benefício: facilita consultas e agregações por vendedor.

3. { price: 1 }
Motivo: ordenações por preço são frequentes, como ordenar do menor ao maior preço.
Benefício: evita sort em memória e permite que o MongoDB use o índice para ordenar rapidamente.

4. { name: "text", description: "text" }
Motivo: suporte a buscas por palavras-chave no nome ou descrição dos produtos.
Benefício: permite buscas textuais eficientes, como db.products.find({ $text: { $search: "fone bluetooth" } }).

5. { location: "2dsphere" }
Motivo: consultas geoespaciais, como produtos próximos a uma localização.
Benefício: habilita operações $near e $geoWithin para buscas rápidas por localização.

6. { "promotions.inicio": 1, "promotions.fim": 1 }
Motivo: encontrar promoções ativas em um intervalo de tempo.
Benefício: otimiza buscas de promoções vigentes com base na data atual.

Coleção reviews
1. { productId: 1, rating: -1 }
Motivo: mostrar avaliações de um produto, geralmente ordenadas por nota.
Benefício: acelera buscas por produto e ordenação direta por rating.

2. { userId: 1 }
Motivo: usuários podem consultar suas próprias avaliações.
Benefício: torna rápidas consultas de avaliações por usuário.

Coleção orders
1. { buyerId: 1, date: -1 }
Motivo: usuários frequentemente consultam seu histórico de pedidos, ordenado do mais recente ao mais antigo.
Benefício: acelera buscas de pedidos do usuário em ordem cronológica inversa.

2. { "items.productId": 1 }
Motivo: verificar vendas de um produto específico.
Benefício: melhora consultas de relatórios de vendas por produto.

3. { status: 1, date: -1 }
Motivo: filtrar pedidos por status e ordenar por data.
Benefício: torna eficiente consultas administrativas e relatórios operacionais.

4. { "items.productId": 1, date: -1 }
Motivo: usado quando o sistema filtra pedidos por produto e período, como vendas do produto X nos últimos 30 dias.
Benefício: filtra e ordena em um único índice composto, sem custo extra de ordenação.
